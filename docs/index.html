Core Philosophy

The Integrated Code Rules and Memory Bank System is designed to balance task execution efficiency with comprehensive project knowledge. It provides a structured approach to maintaining consistent coding standards and documentation practices while ensuring project continuity across multiple sessions.

Key Concept

This system implements a "task-first" approach that prioritizes immediate task execution while maintaining only the minimal context needed for each step. Additional context is loaded progressively only when needed for subsequent steps.

Key Components

Command System

The system uses a set of standardized commands for task execution, memory management, session management, and code implementation. These commands provide a consistent interface for working with the system.

Knowledge Structure

Information is organized in a tiered structure to optimize loading and usage:

  • Bootstrap Tier: Core system structure, loaded only when essential
  • Critical Tier: Current state relevant to immediate tasks
  • Essential Tier: Reference information loaded only when required
  • Reference Tier: Detailed information accessed only when specifically needed

Documentation Standards

The system includes standardized documentation practices that balance detail with efficiency, focusing on targetted updates rather than comprehensive rewrites.

Tool Usage

All operations use standardized XML-formatted tool specifications for consistency and clarity.
