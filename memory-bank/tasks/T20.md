- `parse-tasks.js` (354 lines) — `tasks.md` + `tasks/T*.md` → `task_items` + `task_dependencies` + `task_subtasks`
- `parse-sessions.js` (198 lines) — `sessions/*.md` → `sessions`
- `parse-session-cache.js` (142 lines) — `session_cache.md` → `session_cache`

These parsers were written for mb-core's v6.10-era format. Cross-project validation (Cloudy workspace, 2026-05-22) revealed they need adaptation for:
- Emoji status normalization (🔄→`in_progress`, ✅→`completed`, ⏸️→`paused`)
- Timezone-aware date parsing (IST = UTC+5:30)
- Variable header structures across workspace evolution
- File modification action extraction
- Edit chunk cross-referencing

T20 is now a dependency for the DB-native workflow backfill path (T21 Phase F). Without adapted parsers, the DB cannot be populated from existing text files, making text-primary → DB-primary transition impossible.
